## 二、项目背景与真实问题

### 1. 个人使用场景

个人用树莓派做学习机、私有服务器、家庭小型服务时，常需要手动检查：

- CPU、内存、磁盘是否过载
- SSH、定时任务等关键服务是否正常
- 系统日志是否频繁报错
- 网络连通性、端口监听状态

### 2. 现有方式痛点

个人日常运维只能手动执行命令：

```bash
top、free、df、ps、ss、grep /var/log/syslog
```

**存在明显问题：**

- 操作繁琐、耗时
- 易遗漏异常，排查效率低
- 无统一记录，无法回溯历史状态
- 无法定时自动巡检，依赖人工

### 3. 项目解决的核心问题

> 为个人用户开发**轻量自动化巡检工具**，一键完成全维度检查、生成报告、支持定时运行，简化个人运维、降低操作门槛、减少人工遗漏。

## 四、核心功能模块

> 共 6 大模块，满足课程技术要求

### 模块 1：系统资源巡检

- CPU 使用率、系统负载
- 内存占用、可用内存
- 磁盘分区使用率、挂载点
- 系统运行时间（uptime）

### 模块 2：进程与服务监控

- CPU / 内存 TOP5 进程
- 关键服务（sshd、cron）状态检测
- 异常进程识别

### 模块 3：日志异常分析

- 读取 `/var/log/syslog`
- 过滤 `error`、`fail`、`warn` 等异常关键词
- 统计高频错误、输出异常汇总

### 模块 4：网络连通性检测

- 公网连通性 ping 测试
- 本地端口监听检查（ss）
- 网络基础状态校验

### 模块 5：个人化报告生成

- 整合所有模块结果
- 结构化文本输出、异常标记
- 报告自动保存，方便个人回溯

### 模块 6：定时自动巡检

- crontab 配置每日定时任务
- 自动生成并保存每日报告
- 支持自定义巡检频率

## 八、测试方案

### 1. 功能测试

- 验证 CPU / 内存 / 磁盘数据准确性
- 验证关键服务状态检测正确性
- 验证日志异常统计有效性
- 验证网络连通性、端口检测结果

### 2. 异常测试

- 磁盘高使用率告警
- SSH 服务停止时异常提示
- 网络断开时连通性异常反馈
- 日志大量报错时异常汇总

### 3. 边界测试

- 空日志、超大日志文件处理
- 极低 / 极高 CPU 负载识别
- 无监听端口时正常输出结果
